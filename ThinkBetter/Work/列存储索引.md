#columnstore #SQLServer 

**局限性**
#SQLServer/2014  不允许在具有聚集列存储索引的表上创建其他索引。

#sqlServer/2022 已经支持隔离级别为SNAPSHOT的聚集列存储索引。

```sql
select top 5 %%lockres%%  --计算或显示某一行数据在 SQL Server 内部对应的“锁资源标识符”
from  tab
```

聚集列存储索引 要好于 非聚集列存储索引

非聚集列存储索引处理的是**堆表**。在sys.column_store_segments中，会出现一个标识列，来定位。

所以  **Clustered + Nonclustered_Columntore = Clustered_Columnstore**
