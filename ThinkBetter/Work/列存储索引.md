## 基本原理

列存储索引使用 **X-Velocity 内存压缩引擎**，该引擎已被许多 Microsoft 技术所采用，例如 PowerPivot、Analysis Services 的表格模式等。

### 基本技术
**X-Velocity 内存压缩引擎**的基本技术是将可用数据划分为不同的行组（row groups），也称为段（segments）。每个段应包含大约 100 万行（当前版本中，此数字不受参数控制），并且使用某种内部算法对这些行进行编码和压缩。对于某些需要字典的列（例如字符类型），则使用额外的字典编码转换。
**each column is separated into its own groups of the Segments**

编码和压缩完成后，数据段和字典会被转换为_Blob 对象_并存储在 SQL Server 中。底层存储机制仍然是传统的 8K 页。如果某个_Blob_对象跨越多个页面，则会调用常规的 Blob 存储机制。


注意：理解和优化列存储索引的一个重要方面是，数据在**段内没有排序** （这在2022以及2025版本，使用有序聚集列存储索引，已经优化），这将导致我们在接下来的几篇文章中介绍一些技术来提高它们的压缩率和性能。


### 元数据
存储在 `sys.column_store_segments` 中，称之为目录（Directory）；
其中有一个很重要的信息——删除位图（_Delete Bitmap_）；删除位图本质上是一个存储段中已删除行信息的存储单元。它在内存中以位图的形式表示，而在磁盘上则以 B 树的形式存储，其中存储了已删除行的 ID。删除位图会定期被查询，以避免返回已删除的行。
