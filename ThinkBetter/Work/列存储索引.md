#columnstore #SQLServer 

**局限性**



列存储索引使用 **X-Velocity 内存压缩引擎**，该引擎已被许多 Microsoft 技术所采用，例如 PowerPivot、Analysis Services 的表格模式等。

该引擎的基本技术是将可用数据划分为不同的行组（row groups），也称为段（segments）。每个段应包含大约 100 万行（当前版本中，此数字不受参数控制），并且使用某种内部算法对这些行进行编码和压缩。对于某些需要字典的列（例如字符类型），则使用额外的字典编码转换。
**each column is separated into its own groups of the Segments**

编码和压缩完成后，数据段和字典会被转换为_Blob 对象_并存储在 SQL Server 中。底层存储机制仍然是传统的 8K 页。如果某个_Blob_对象跨越多个页面，则会调用常规的 Blob 存储机制。
